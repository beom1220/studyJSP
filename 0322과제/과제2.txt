질문 : Oracle 데이터베이스에서 char과 varchar2의 차이점에 대해 설명해줄래?

답변 :
Oracle 데이터베이스에서 CHAR와 VARCHAR2는 문자 데이터를 저장하는 데 사용되는 두 가지 유형의 데이터 타입입니다. 주요 차이점은 저장 공간 할당 방식과 저장할 수 있는 데이터의 길이에 있습니다.

CHAR:

CHAR 타입은 고정 길이 문자 데이터를 저장합니다. 선언할 때 지정된 길이로 공간을 할당하고, 저장된 문자열이 선언된 길이보다 짧은 경우 나머지 공간을 공백으로 채웁니다.
예를 들어, CHAR(10)으로 선언된 필드에 'ABC'를 저장하면 나머지 7자리는 공백으로 채워집니다.
일반적으로 고정 길이의 데이터를 저장할 때 사용되며, 저장되는 데이터의 크기가 항상 일정할 때 유용합니다.
VARCHAR2:

VARCHAR2 타입은 가변 길이 문자 데이터를 저장합니다. 저장된 데이터의 실제 길이만큼만 공간을 할당하며, 선언된 길이까지 동적으로 데이터를 저장할 수 있습니다.
예를 들어, VARCHAR2(10)으로 선언된 필드에 'ABC'를 저장하면 실제로 3자리만 사용되고, 나머지 공간은 할당되지 않습니다.
대부분의 경우에 VARCHAR2를 사용하는 것이 좋으며, 데이터의 길이가 변할 수 있는 필드에 적합합니다.
요약: CHAR은 고정 길이 데이터에 적합하며, 선언 길이에 미치지 못하는 데이터에 대해 공백으로 채워 저장 공간을 낭비할 수 있습니다. 반면, VARCHAR2는 가변 길이 데이터에 적합하며, 실제 데이터 길이만큼만 공간을 사용하여 저장 공간을 보다 효율적으로 사용합니다. 데이터의 길이가 미리 예측 가능하고 항상 일정한 경우 CHAR을, 그렇지 않은 경우에는 VARCHAR2를 사용하는 것이 좋습니다.
